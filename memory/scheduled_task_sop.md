# Scheduled Tasks SOP

Directory: `../sche_tasks/` stores task definition JSON files, `../sche_tasks/done/` stores execution reports.

## Task JSON Format (*.json)
```json
{"schedule":"08:00", "repeat":"daily", "enabled":true, "prompt":"...", "max_delay_hours":6}
```
Available `repeat` values: daily | weekday | weekly | monthly | once | every_Nh (every N hours) | every_Nd (every N days)
`max_delay_hours` (optional, default 6): Hours after `schedule` past which the task won't trigger, preventing late boot-ups from running outdated tasks.

## Trigger Flow
1. `scheduler.py` (inside `reflect/`) polls `sche_tasks/*.json` every 60 seconds.
2. Triggers only if all conditions are met: `enabled=true` + current time ≥ `schedule` + cooldown time passed (based on the latest report timestamp in `done/`).
3. Upon triggering, it assembles a `prompt` including the report path `../sche_tasks/done/YYYY-MM-DD_TaskName.md`.
4. **First thing to do upon receiving a task**: Use `update_working_checkpoint` to log the target report file path, preventing forgetting it during long task execution.
5. After execution, write a report to the above path (the scheduler relies on this file to verify it has executed today).

## Logging & Monitoring
- Scheduler automatically writes logs to `sche_tasks/scheduler.log` (trigger/skip/error).
- `scheduler.health_check()` returns a status list of all tasks (HEALTHY/OVERDUE/DISABLED/NEVER_RUN/ERROR).
- JSON parsing errors, format errors in schedule, and unknown repeat types will be logged.

## Note
- `once` type: Cooldowns for 100 years after executing once (effectively skipping permanently).
- Task files only dictate "what to do"; the report path is auto-generated by the scheduler and injected into the prompt.
- The `sche_tasks` directory is under `../` (which is the code root).
